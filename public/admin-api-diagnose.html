🔍 API 分頁診斷工具

偵測你的託管環境 API 真正能回傳多少筆資料、每頁上限是多少，以及搜尋功能是否能找到整個資料庫的資料。
在你的託管空間執行此頁面，比較和本機開發環境的差異。

📊 Step 1：API 基本資訊偵測

測試 tables/game_database 的 total、實際回傳筆數、以及不同 limit 值的行為。

📦 Step 2：分批載入完整性測試

模擬 game-names.js 的分批載入邏輯（先抓 total，再分頁 page=1,2,3...）， 確認是否能抓到全部資料、有無 missing pages、重複資料。

🔎 Step 3：搜尋功能測試

測試 ?search= 能不能找到資料庫後段的遊戲（例如大西部之路、約克公爵…）。 如果搜尋找不到但分批能找到，代表 API 的 search 有資料範圍限制。

📄 Step 4：指定分頁內容查看

直接查看某一頁的資料，確認第幾頁開始出現 "大西部之路"。

🗺️ Step 5：GameNames 快取診斷

測試 GameNames.load() 實際載入幾筆、localStorage 快取狀態，以及能否查到 "大西部之路"。
